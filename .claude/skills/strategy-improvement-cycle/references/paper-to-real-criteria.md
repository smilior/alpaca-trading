# ペーパーからリアルへの移行基準 リファレンス

## なぜ移行基準が必要か

ペーパートレーディングで利益が出ている ≠ リアルで利益が出る。

理由:
1. ペーパーにはスリッページがほぼない
2. ペーパーでは全注文が即座に約定する
3. ペーパーでは感情（恐怖、欲）が関与しない
4. ペーパーの期間が短いと統計的に無意味

**だから厳格な移行基準を設け、基準を満たすまで移行してはならない。**

## 移行基準チェックリスト

### 定量基準（すべて満たすこと）

#### 1. 最低取引回数: 100回以上

```
なぜ100回か:
- 30回未満: 統計的に何も言えない（中心極限定理の最低ライン未満）
- 30-100回: 大まかな傾向は見えるが、信頼区間が広い
- 100回以上: 平均リターンの推定が安定し始める
- 300回以上: 理想的（ただし時間がかかりすぎる場合は100回で可）

計算例:
- 日次1-2回の取引 → 50-100営業日（2-5ヶ月）
- 週次1-2回の取引 → 50-100週（1-2年）← 非現実的
→ 取引頻度が低い戦略は、バックテストで補完する
```

#### 2. 最低運用期間: 3ヶ月以上

```
なぜ3ヶ月か:
- 1ヶ月: たまたま市場が上昇/下落しただけかもしれない
- 2ヶ月: やや長いが、1つの市場環境しか経験していない可能性
- 3ヶ月以上: 異なる市場環境を経験する確率が高まる
- 理想: 6ヶ月以上（ブル/ベア/レンジを全て経験）

重要: 3ヶ月間ずっと上昇相場だった場合、
ベア相場のデータがないことに注意。
```

#### 3. シャープレシオ > 1.0

```
年率化シャープレシオ:
- < 0: 論外。リスクフリーレート以下
- 0-0.5: 不十分。SPYのBuy&Hold（約0.5-0.8）と変わらない
- 0.5-1.0: 境界線。リアルではスリッページで0.2-0.3下がる可能性
- 1.0-2.0: 合格。リアルでもプラスを維持できる可能性が高い
- > 2.0: 良好だが、過学習を再確認すること

注意: ペーパーのシャープレシオからリアルのシャープレシオを
推定するには、0.2-0.3を引くのが保守的な見積もり。
ペーパーで1.2 → リアルでは0.9-1.0程度と想定。
```

#### 4. 最大ドローダウン < 15%

```
なぜ15%か:
- ペーパーのMDD × 1.5-2.0 = リアルの想定MDD
- ペーパーで15% → リアルでは22-30%の可能性
- 30%のMDDは心理的に非常にきつい
  （$100,000 → $70,000。回復に43%のリターンが必要）
- 15%以下なら、リアルでも20%前後に収まり、回復可能
```

#### 5. プロフィットファクター > 1.5

```
なぜ1.5か:
- 1.0-1.2: ペーパーではギリギリプラスだが、
  リアルのコスト（スリッページ、スプレッド拡大）で
  マイナスになる可能性が高い
- 1.2-1.5: 境界線。リアルでは1.0前後（トントン）の可能性
- 1.5以上: リアルでも1.2以上を維持できる可能性が高い
```

#### 6. ベンチマーク（S&P500）を上回っていること

```
なぜ必要か:
SPYのBuy&Holdに負けるなら、
わざわざ複雑なアルゴ取引をする意味がない。
SPYを買って放置した方がコスト（時間・API費用）がかからない。

基準:
- 3ヶ月間の累積リターン > SPYの同期間リターン
- リスク調整後（シャープレシオ）でもSPYを上回ること
```

#### 7. 異なる市場環境を経験していること

```
理想的には:
- 上昇相場: 市場全体が+5%以上の月を含む
- 下落相場: 市場全体が-5%以上下落した月を含む
- 横ばい相場: 市場全体の変動が±2%以内の月を含む

3つ全てを経験していない場合:
- 未経験の環境についてはバックテストで補完
- リアル移行後も小さいポジションで慎重に運用
```

### 定性基準

- [ ] 戦略の仮説が明確に言語化されている
- [ ] リスク管理ルールが全て実装・テスト済み
- [ ] エラーハンドリングが実装・テスト済み
- [ ] 回路ブレーカーが実装・テスト済み
- [ ] APIキーの管理が安全に行われている
- [ ] ログ管理・アラート通知が実装済み
- [ ] 緊急時の手動操作手順が文書化されている

## 移行後の段階的な資金投入計画

**移行基準を満たしても、一度に全資金を投入してはならない。**

### 段階的投入スケジュール

```
Phase 1: 最小額テスト（1-2週間）
- 投入額: 計画額の10%
- 目的: リアル環境での挙動確認
- 確認項目:
  - 約定価格とペーパーの差（スリッページ測定）
  - 注文の約定速度
  - エラー発生率
  - 感情的な影響の確認

Phase 2: 小額運用（2-4週間）
- 投入額: 計画額の25%
- 目的: リアルでのKPI測定
- 確認項目:
  - シャープレシオがペーパーと大きく乖離していないか
  - ドローダウンが想定範囲内か
  - スリッページの実測値

Phase 3: 中額運用（4-8週間）
- 投入額: 計画額の50%
- 目的: 安定運用の確認
- 確認項目:
  - 前Phaseと同じKPIが維持されているか
  - 市場インパクトが発生していないか

Phase 4: 全額運用
- 投入額: 計画額の100%
- 条件: Phase 1-3で問題がなかった場合のみ
```

### 各フェーズの中止基準

```
Phase中止基準:
- 日次損失が投入額の5%を超えた
- シャープレシオがペーパー実績の50%を下回った
- 想定外のエラーが3回以上発生した
- スリッページが想定の2倍以上

中止後のアクション:
1. 全ポジションクローズ
2. 原因分析（なぜペーパーと乖離したか）
3. 対策を実施
4. 前のPhaseからやり直し
```
