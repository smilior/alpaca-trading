# ディベートフォーマット リファレンス

## 1. ソクラテス式問答法

### 概要
質問を繰り返すことで、前提の矛盾や隠れた仮定を炙り出す手法。

### 使うタイミング
- 「これは当然正しい」と全員が思い込んでいる前提を疑いたいとき
- 議論の前提条件を明確にしたいとき

### 質問テンプレート

```
1. 明確化の質問
   「〜とはどういう意味か？具体的に何を指しているか？」
   例: 「AIトレーディングが有効とはどういう意味か？
        何と比較して有効なのか？」

2. 前提を問う質問
   「なぜそう思うのか？その前提は何か？」
   例: 「過去のバックテスト結果が将来も有効だと
        なぜ言えるのか？」

3. 根拠を問う質問
   「それを支持するデータは何か？」
   例: 「センチメント分析が有効だというデータは
        どの期間・どの市場で得られたものか？」

4. 反例を問う質問
   「もし〜でなかったらどうなるか？」
   例: 「もしニュースのセンチメントと株価が
        逆相関する局面があったらどうなるか？」

5. 帰結を問う質問
   「それが正しいとすると、何が起きるはずか？」
   例: 「この戦略が本当に有効なら、
        なぜ既に誰もがやっていないのか？」
```

### トレーディングでの適用例

```
命題: 「RSI < 30 で買い、RSI > 70 で売りは有効」

ソクラテス式質問:
Q: 「有効とはどういう意味か？」
A: 「S&P500のBuy & Holdを上回る」

Q: 「どの期間で検証したか？」
A: 「2020-2024年」

Q: 「2022年のベア相場ではどうだったか？」
A: 「...確認していない」

Q: 「RSI < 30 が有効でないケースはどんなときか？」
A: 「トレンド相場で連続的に下落するとき」

Q: 「つまりRSIはレンジ相場でのみ有効であり、
    レンジ/トレンドの判別なしに使うのは危険では？」
A: 「...確かに、市場状態の判別が前提条件として必要」

→ 結果: RSI単体での売買は不十分。市場レジーム判別との
  組み合わせが必要という結論に到達
```

## 2. 赤チーム・青チーム方式

### 概要
- **青チーム**: 計画の推進側。戦略の有効性を主張
- **赤チーム**: 攻撃側。戦略の脆弱性を探す

### 使うタイミング
- 戦略をリアルマネーに移行する前の最終検証
- 重大な意思決定の前

### 進行手順

```
Phase 1: 青チームが戦略を提示（15分）
- 戦略の概要
- バックテスト結果
- リスク管理方針
- 期待されるパフォーマンス

Phase 2: 赤チームが攻撃（20分）
- 過学習の可能性を検証
- エッジケース（暴落、フラッシュクラッシュ）での挙動
- データの信頼性を疑う
- 隠れた相関・バイアスを指摘
- 「この戦略を壊す方法」を提案

Phase 3: 青チームが防御（10分）
- 赤チームの指摘への回答
- 対策の提示

Phase 4: 判定（5分）
- 赤チームが突破できなかった点 → 戦略の強み
- 赤チームが突破した点 → 修正が必要
- 判定: Go / No-Go / 条件付きGo
```

## 3. Delphi法（デルファイ法）

### 概要
匿名で意見を出し、集約し、再投票するプロセスを繰り返す。グループシンクを防ぐ。

### 使うタイミング
- 不確実性が高い予測（市場の将来動向等）
- 権威バイアスを排除したいとき

### 進行手順

```
Round 1: 匿名投票
- 各エージェントが独立に予測・意見を提出
- 例: 「来月のS&P500の方向性は？」
  - Agent A: 上昇（根拠: ...）
  - Agent B: 横ばい（根拠: ...）
  - Agent C: 下落（根拠: ...）

Round 2: 結果共有と議論
- 全投票結果を匿名で共有
- 外れ値（極端な意見）の根拠を重点的に議論
- 各自の前提条件を明確化

Round 3: 再投票
- 議論を踏まえて再度投票
- 収束度合いを確認
- 収束しない場合は、分岐点（何が意見を分けているか）を特定

最終判定:
- 収束した場合 → その結論を採用
- 収束しない場合 → 各シナリオの確率を割り当てる
  例: 上昇60% / 横ばい25% / 下落15%
  → シナリオ別のポジション設計に使う
```

### AIエージェントでの実装

AIエージェント間でDelphi法を実施する場合、「匿名性」の代わりに「独立性」を確保する：
- 各エージェントに個別にプロンプトを送り、他のエージェントの回答を見せない
- Round 2で初めて全回答を共有する

## 4. 6 Thinking Hats（de Bono）

### 概要
Edward de Bonoが提唱した並行思考法。6つの「帽子」を順番にかぶることで、異なる視点から網羅的に議論する。

### 6つの帽子

| 帽子 | 視点 | 問い | トレーディングでの例 |
|------|------|------|---------------------|
| 白（事実） | データ・事実のみ | 何がわかっているか？何がわからないか？ | バックテスト結果、市場データ、取引コスト |
| 赤（感情） | 直感・感覚 | どう感じるか？（根拠不要） | この戦略に自信があるか？不安はないか？ |
| 黒（批判） | リスク・問題点 | 何が問題か？なぜうまくいかないか？ | オーバーフィッティングのリスク、市場変動 |
| 黄（楽観） | メリット・可能性 | なぜうまくいくか？最良のケースは？ | LLMの言語理解力、自動化のメリット |
| 緑（創造） | 新しいアイデア | 別のやり方はないか？ | 代替戦略、新しいデータソース |
| 青（管理） | プロセス管理 | 議論をどう進めるか？何が決まったか？ | 議事進行、結論の整理 |

### 使うタイミング
- ブレインストーミング段階で、アイデアを広げたいとき
- チーム内で議論が一方向に偏っていると感じるとき

### 進行手順

```
1. 青帽子: 今日の議論のゴールを確認
2. 白帽子: 手元にあるデータ・事実を整理
3. 緑帽子: 自由にアイデアを出す
4. 黄帽子: 各アイデアのメリットを分析
5. 黒帽子: 各アイデアのリスク・問題点を分析
6. 赤帽子: 直感的にどう感じるか（根拠不要）
7. 青帽子: 結論をまとめ、次のアクションを決定
```

**ポイント**: 全員が同じ帽子を同時にかぶる。「黒帽子の時間」に楽観論を述べてはいけない。これにより対立ではなく並行思考が実現する。

## 各手法の使い分けガイド

| 状況 | 推奨手法 |
|------|----------|
| 前提を疑いたい | ソクラテス式問答法 |
| 戦略の最終検証 | 赤チーム・青チーム |
| 不確実性が高い予測 | Delphi法 |
| 幅広いアイデア出し | 6 Thinking Hats |
| 二者択一の判断 | 構造化ディベート（SKILL.md） |
| 複数案の比較 | 構造化ディベート × 赤/青チーム |

## 組み合わせパターン

複雑な意思決定では、複数の手法を組み合わせよ：

```
例: 新しいトレーディング戦略の採否を決める

1. 6 Thinking Hats でアイデアを広げる（発散フェーズ）
2. 構造化ディベートで2-3案に絞る（収束フェーズ）
3. 赤/青チームで最有力案を攻撃テスト（検証フェーズ）
4. Delphi法で最終判定（決定フェーズ）
```
