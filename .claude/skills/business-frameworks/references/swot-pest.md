# SWOT分析 / PEST分析 リファレンス

## SWOT分析

### 概要

内部環境（Strengths/Weaknesses）と外部環境（Opportunities/Threats）を整理し、戦略の方向性を導出するフレームワーク。

### 4象限の書き方

| | プラス要因 | マイナス要因 |
|---|---|---|
| **内部環境** | Strengths（強み） | Weaknesses（弱み） |
| **外部環境** | Opportunities（機会） | Threats（脅威） |

**内部環境**: 自分たちがコントロールできるもの
**外部環境**: 自分たちではコントロールできないもの

### トレーディングボットでのSWOT例

#### Strengths（強み）
```
✅ 良い書き方（具体的・定量的）:
- Claude APIにより決算レポートの自然言語分析が可能
  （400ページのレポートを5分で処理）
- Alpaca APIで手数料無料の自動売買が可能
- Python + SQLiteで軽量なシステム構築が可能
- ペーパートレーディングで無リスクに検証可能

❌ 悪い書き方（漠然・定性的）:
- AIが使える
- プログラミングができる
```

#### Weaknesses（弱み）
```
✅ 良い書き方:
- リアルタイムデータは無料プランでは15分遅延
- バックテスト期間が短い（十分なデータ蓄積がない）
- 高頻度取引はレイテンシの制約で不可能
- LLMの推論コスト（1呼び出し$0.01-0.10）が累積する
- 運用経験がなく、想定外の市場状況への対応が未知数

❌ 悪い書き方:
- まだ初心者
- お金が少ない
```

#### Opportunities（機会）
```
✅ 良い書き方:
- LLMの性能向上が続いており、分析精度が自然に改善される
- Alpaca APIの機能拡充（オプション取引対応等）
- AIトレーディングはまだ個人レベルでは普及途上
- 米国市場は世界最大の流動性を持ち、アルゴ取引に適している

❌ 悪い書き方:
- AIが流行っている
- 株価は上がるかもしれない
```

#### Threats（脅威）
```
✅ 良い書き方:
- 機関投資家のアルゴ取引は遥かに高速・高度
  （ミリ秒単位の競争では勝てない）
- SEC等による個人アルゴ取引の規制強化リスク
- Claude APIの価格改定リスク（コスト構造の変化）
- 市場レジームの変化（量的引き締め→低流動性）
- ブラックスワンイベント（パンデミック、戦争等）

❌ 悪い書き方:
- 市場は不確実
- 競合が強い
```

### クロスSWOT戦略の導出

SWOT分析の真の価値はクロス分析にある。4象限を組み合わせて戦略を導出せよ。

| | Strengths | Weaknesses |
|---|---|---|
| **Opportunities** | **SO戦略**（攻め） | **WO戦略**（改善） |
| **Threats** | **ST戦略**（差別化） | **WT戦略**（防御/撤退） |

#### SO戦略（強み × 機会）→ 積極的に攻める

```
例: Claude APIの分析力（S）× LLM性能向上（O）
→ 決算発表直後の自動分析による情報優位性を最大化する戦略
  （機関投資家が使わないLLM分析という独自の切り口）
```

#### WO戦略（弱み × 機会）→ 弱みを機会で補う

```
例: リアルタイムデータ遅延（W）× 米国市場の流動性（O）
→ 高頻度ではなく、日次・週次の中長期戦略に特化する
  （15分遅延が問題にならない時間軸を選ぶ）
```

#### ST戦略（強み × 脅威）→ 脅威に強みで対抗

```
例: LLM自然言語分析（S）× 機関投資家の高速アルゴ（T）
→ 速度で競争せず、LLMの言語理解力で差別化する
  （テクニカル分析は機関に勝てない → ファンダメンタルズ+ニュース分析で勝負）
```

#### WT戦略（弱み × 脅威）→ 損失最小化・撤退計画

```
例: 運用経験不足（W）× ブラックスワン（T）
→ 最大ドローダウン15%で全ポジション強制クローズするルール
  想定外の事態では「何もしない」ことが最善策になりうる
```

## PEST分析

### 概要

マクロ外部環境を Political（政治）、Economic（経済）、Social（社会）、Technological（技術）の4軸で分析する。SWOT分析の「外部環境」部分をより体系的に分析するために使う。

### 4軸の分析

#### Political（政治・規制）
- SEC（米国証券取引委員会）のアルゴ取引規制動向
- PDT（Pattern Day Trader）ルール：口座残高$25,000未満で日計り制限
- 税制：キャピタルゲイン税率（短期 vs 長期）
- 日米租税条約に基づく源泉徴収（日本居住者の場合）
- AI規制の動向（EU AI Act等の影響波及）

#### Economic（経済）
- FRBの金利政策（利上げ/利下げサイクル）
- インフレ率とその市場への影響
- 為替（USD/JPY）の変動（日本円ベースでのリターン計算）
- リセッション確率と市場全体のリスクオン/オフ
- クレジットスプレッド（信用リスクの指標）

#### Social（社会）
- ミーム株現象（Reddit, GameStop等）の影響
- AI・ロボアドバイザーの社会的受容度
- 個人投資家の市場参加率の変化
- ESG投資の潮流

#### Technological（技術）
- LLM（大規模言語モデル）の性能向上速度
- APIインフラの安定性・コスト低下傾向
- クラウドコンピューティングの価格動向
- リアルタイムデータフィードの品質向上
- 量子コンピューティング（長期的な脅威）

### PEST → SWOT への統合

PEST分析の結果は、SWOT分析の「機会（O）」と「脅威（T）」に直接マッピングする：

```
PEST分析結果 → SWOTのO/Tに分類：

P: PDTルールで日計り制限 → T（脅威）
P: AI規制はまだ緩い → O（機会）
E: 利下げサイクル → O（株高の追い風）
E: 円安リスク → T（為替損失リスク）
S: 個人投資家増加 → O（流動性向上）
T: LLM性能向上 → O（分析精度向上）
T: 機関投資家のAI高度化 → T（競争激化）
```

この統合により、SWOT分析の外部環境部分がPESTで裏打ちされ、より体系的な戦略策定が可能になる。
